This topic describes the entity and type representations for common items that the Volunteer API uses. To learn about how to set up your developer account and work with the SKY API, see the Getting Started guide.

  • Assignment
  • Availability

Assignment

The assignment entity has the following properties:

Property Type Description
constituent_id string The constituent identifier for the volunteer.
job_name string The name of the assigned job.
location string The location of the assignment.
department string The department of the assignment.
start dateTime The start date of the assignment. Uses ISO-8601 format: 1969-11-21T10:29:43.
end dateTime The end date of the assignment. Uses ISO-8601 format: 1969-11-21T10:29:43.

Availability

The availability entity has the following properties:

Property Type Description
days string The days when the volunteer is available. Available values are sundays, mondays, tuesdays, wednesdays, thursdays, fridays, saturdays, alldays, weekdays, weekends, and notspecified.
from fuzzy date The date when the volunteer becomes available.
to fuzzy date The date when the volunteer ceases to be available.
start_time timespan The time when the volunteer becomes available.
end_time timespan The time when the volunteer ceases to be available.

Summary

The summary entity has the following properties:

Property Type Description
first_year_volunteered integer This computed field retrieves the volunteer's first year volunteered.
total_hours decimal This computed field retrieves the volunteer's total hours volunteered.
total_value currency This computed field calculates the estimated monetary value of the work that the volunteer has completed.
total_years_volunteered integer This computed field calculates the volunteer's total years volunteered.
first_assignment assignment This computed field retrieves the volunteer's first assignment.
last_assignment assignment This computed field retrieves the volunteer's most recent assignment.
